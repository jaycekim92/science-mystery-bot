# "그거 왜 그래?" 본문 작성 가이드

루틴(에이전트)이 `pick_topic.py`가 준 소재 JSON을 받아, 이 가이드대로 **본문 텍스트**를 작성한다.
카드 이미지는 `generate_card.py`가 자동 생성하므로, 여기서는 **피드 본문 글만** 쓴다.

## 컨셉
누구나 겪지만 설명 못 하는 **일상 미스터리 과학**. 목표는 "오 신기"에서 끝나지 않고 **댓글이 달리게** 하는 것.

## 3단 구조 (이 순서로)
1. **공감 후킹** — "~한 적 있죠", "다들 있죠"처럼 독자가 "맞아 나도!" 하며 멈추게.
2. **과학 설명** — 소재 JSON의 `explanation`을 자연스러운 구어체로. 정설/가설을 구분해 전달.
3. **열린 질문** — 소재 JSON의 `closing_question`을 살려 댓글을 유도. (경험소환/추측초대/편가르기/통념반전)
4. 마지막 줄: `📚 {source}` (출처 1줄)

## 톤 규칙 (반드시)
- **단정 금지.** 정설(`status=정설`)만 단정 가능. 가설은 **"가장 유력한 설명은 ~"** 로 긍정 제시.
- ⚠️ **"사실 과학자들도 아직 잘 몰라요" 같은 김빠지는 표현 금지.** 유저가 "결론 없네"로 느낌.
  → 알려진 것까지 확실히 주고, 모르는 부분은 굳이 언급 말고 열린 질문으로 닫는다.
- **통념반전**("~인 줄 알았는데 연구에선 안 나왔어요")은 '모른다'가 아니라 **명확한 정보**라 적극 활용.
- 존댓말 + 친근한 구어체. 과장·낚시 금지. 분량 3~5문장 + 질문.

## 출력 형식
본문 텍스트만. 작성 후 다음을 실행:
- `python3 generate_card.py {id}`
- `python3 notify_slack.py {id} --body "{작성한 본문}"`

## 예시 (id=5, 통념반전)
```
녹음된 내 목소리 처음 듣고 "이게 내 목소리라고?" 충격받은 적 있죠.

평소 내가 듣는 목소리는 공기로 전해지는 소리 + 두개골 뼈로 울리는 소리가 합쳐진 거예요. 뼈가 저음을 보태줘서 더 굵게 들리는 거죠. 근데 녹음기엔 공기로 간 소리만 잡혀요. 그게 바로 남들이 매일 듣는 진짜 내 목소리예요.

남이 듣는 목소리 vs 내가 듣는 목소리, 어느 쪽이 '진짜'일까요?

📚 Royal Society Open Science (2023)
```
